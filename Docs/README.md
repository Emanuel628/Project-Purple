# Project Purple Docs

This folder contains the planning and product documentation for Project Purple.

## Documents

- `BUILD_PLAN.md` — high-level build plan and first milestone scope
- `BRAND_DIRECTION.md` — brand identity, language, and layout direction
- `DESIGN_SYSTEM.md` — design tokens, UI principles, components, and layout rules
- `PAGE_MAP.md` — app pages, routes, sections, and page responsibilities
- `PHASES.md` — phased build strategy and scope control
- `DATA_MODEL_DRAFT.md` — future-facing data model draft for backend planning

## Documentation Rules

- Keep documentation current when major decisions change.
- Do not mix active task tracking into core docs; use `Work to do/`, `Work to review/`, and `Completed work/` for workflow tracking.
- Keep Phase 1 documentation focused on the static app shell.
- Mark future backend/video/payment notes clearly as future-phase work.
