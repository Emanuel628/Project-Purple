# Project Purple

Project Purple is the working development name for a premium creator-first video platform.

The goal is to build a clean, modern video app shell with light and dark themes, purple branding, creator-centered discovery, collections, and a cinematic watch experience that does not copy the standard YouTube layout.

This first phase focuses only on the static app shell, design system, page structure, and mock data. Backend services, authentication, real video uploads, payments, and streaming infrastructure will be added in later phases.

## Core Direction

Project Purple should feel:

- Premium, polished, and creator-first
- Light-mode friendly with an elegant dark mode
- Purple, silver, black, and white as the main visual identity
- More like a curated creator platform than a generic video warehouse
- Built around creators, collections, discovery, and watch paths

## Phase 1 Scope

Phase 1 is only the app shell.

Build:

- Static page structure
- Light and dark design tokens
- App layout foundation
- Reusable UI components
- Mock creator and video data
- Landing page
- Home / discovery page
- Watch page shell
- Creator page shell
- Collections page shell
- Studio shell
- Upload shell
- Vault / saved content shell
- Settings shell

Do not build yet:

- Real authentication
- Real video upload
- Video transcoding
- Database models
- Payments
- Creator payouts
- Recommendation algorithms
- Livestreaming
- Mobile apps

## Product Language

Use Project Purple language instead of copying YouTube wording.

Preferred terms:

- Discover instead of Explore
- Creator Page instead of Channel
- Collections instead of Playlists
- Watch Path instead of Up Next
- Vault instead of Library
- Rising Creators instead of Trending
- Studio instead of Creator Studio

## Build Phases

1. Static app shell
2. Frontend interactions and mock data
3. Auth and backend foundation
4. Video upload and streaming pipeline
5. Creator Studio tools
6. Monetization
7. Moderation, trust, and scale
