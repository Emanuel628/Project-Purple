# Contributing to Project Purple

Project Purple is being built in structured phases.

## Build Discipline

Do not add future-phase systems early.

Current phase rules:

- Phase 1 is static app shell only.
- Do not add backend code during Phase 1.
- Do not add authentication during Phase 1.
- Do not add real video upload during Phase 1.
- Do not add payments during Phase 1.
- Do not copy YouTube's standard layout.

## Code Organization

Keep files in their proper folders:

```txt
Docs/             Project documentation
public/           Static shell files
src/              Future source code
Work to do/       Planned tasks
Work to review/   Pending review notes
Completed work/   Finished work records
```

## Pull Request Expectations

Every pull request should include:

- What changed
- Why it changed
- What phase it belongs to
- Screenshots for UI work when possible
- Confirmation that no out-of-scope systems were added

## Design Expectations

Project Purple should remain:

- Premium
- Creator-first
- Purple branded
- Light and dark mode ready
- Organized
- Distinct from YouTube
